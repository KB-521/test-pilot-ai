# 异常处理与错误码规范

## 文档说明

- 覆盖规则：异常分类、错误码设计、抛错策略和外部响应映射。
- 核心要求：内部异常与外部错误语义分层；禁止把原始堆栈或底层错误直接暴露给客户端。
- 应用场景：API 实现、service 设计、第三方调用、批处理任务。
- 命中主题：异常、错误码、错误语义
- 触发关键词：exception,error code,HTTPException,业务异常
- 关联 skill：python/backend-design、python/req-impl、python/code-review
- 前置规则：06、07、12
- 后置规则：09、22、27
- 排除场景：纯文档维护

## 规则

1. 参数错误、权限错误、资源不存在、业务冲突要分开处理。
2. 底层异常先记录日志，再映射为稳定的业务错误。
3. 对外错误码和消息要可预测、可文档化。
4. 缺陷修复若涉及错误语义变化，必须同步更新 API 文档。
