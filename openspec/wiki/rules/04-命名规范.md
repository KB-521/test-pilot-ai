# 命名规范

## 文档说明

- 覆盖规则：模块、文件、类、函数、常量、API 路径和变更 ID 的命名规范。
- 核心要求：命名必须稳定、可检索、与职责一致，禁止语义模糊和随意缩写。
- 应用场景：创建文件、接口、模块、需求与设计文档时。
- 命中主题：命名、可读性、可检索性
- 触发关键词：命名,rename,path naming,scope
- 关联 skill：common/init-agent、python/backend-design、python/req-impl、web/task-impl
- 前置规则：03
- 后置规则：06、21、26
- 排除场景：纯 review 结论汇总

## 规则

1. 文件和目录优先使用 `kebab-case` 或项目既有约定。
2. Python 模块、函数、变量使用 `snake_case`。
3. 类名使用 `PascalCase`。
4. 常量使用 `UPPER_SNAKE_CASE`。
5. API 资源路径统一小写，优先 `snake_case`。
6. `change-id` 必须稳定，不能用临时描述性句子。
