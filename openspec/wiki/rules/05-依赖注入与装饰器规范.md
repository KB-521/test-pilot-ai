# 依赖注入与装饰器规范

## 文档说明

- 覆盖规则：FastAPI 依赖注入、Python 装饰器与中间件的使用边界。
- 核心要求：依赖注入只承载横切关注点和上下文装配；业务逻辑不能藏进装饰器里。
- 应用场景：设计 FastAPI 路由、接入认证、日志、上下文或通用校验能力。
- 命中主题：Depends、middleware、decorator、横切关注点
- 触发关键词：Depends,middleware,decorator,依赖注入,鉴权
- 关联 skill：python/backend-design、python/req-impl、python/code-review
- 前置规则：02、06
- 后置规则：09、12、18
- 排除场景：纯数据模型设计

## 规则

1. FastAPI `Depends` 用于鉴权、上下文、配置和通用资源注入。
2. 业务规则不要封装进看不见的装饰器链。
3. 中间件只处理横切逻辑，如 request id、日志、trace、通用异常转换。
4. 任何会影响请求语义的装饰器，都要在技术文档中说明。
