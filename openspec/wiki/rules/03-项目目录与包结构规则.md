# 项目目录与包结构规则

## 文档说明

- 覆盖规则：Python 项目、前端项目和混合仓库的目录组织建议。
- 核心要求：目录结构必须服务协作与分层，不得把业务逻辑、配置和测试随意混堆。
- 应用场景：初始化仓库、整理目录、生成 `AGENTS.md`。
- 命中主题：目录结构、包结构、仓库组织
- 触发关键词：目录结构,package layout,src,tests,apps
- 关联 skill：common/init-agent、common/init-tech、common/init-business
- 前置规则：02
- 后置规则：04、23
- 排除场景：只补一条局部文档

## Python 后端推荐结构

```text
backend/
├── app/
│   ├── routers/
│   ├── services/
│   ├── repositories/
│   ├── schemas/
│   ├── domain/
│   └── common/
├── tests/
│   ├── api/
│   └── unit/
└── pyproject.toml
```

## 规则

1. `tests/` 与生产代码分开。
2. 路由、service、repository 分目录，不混在一个包里。
3. 配置与密钥入口集中，避免散落。
4. 公共工具放 `common` 或 `shared`，不要混入业务目录。
