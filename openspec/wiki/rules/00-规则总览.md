# OpenSpec 规则总览

## 文档说明

- 覆盖规则：规则库目标、默认技术栈口径、阅读顺序与执行链路。
- 核心要求：所有实现任务必须先读索引，再命中规则正文；默认 Python 栈只作为建议，不得冒充仓库事实。
- 应用场景：初始化规则库、实现前装载规则、评审交付是否满足门禁。
- 命中主题：规则总览、阅读顺序、默认基线、执行链路
- 触发关键词：规则总览,先看哪些规则,默认栈,门禁,实现前准备
- 关联 skill：req-dev、common/init-rules、python/req-impl、web/task-impl
- 前置规则：无
- 后置规则：01、06、22、27
- 排除场景：单纯业务讨论、无技术决策的闲聊

## 默认后端建议

- Python `3.12`
- FastAPI
- Pydantic v2
- SQLAlchemy 2 + Alembic
- pytest + httpx
- Ruff

## 规则读取顺序

1. 先读 `INDEX.md`。
2. 再读本文件确认整体执行链路。
3. 初始化任务优先看：`01`、`02`、`03`、`23`。
4. 实现任务至少继续下钻：`06`、`07`、`08`、`09`、`11`、`12`、`14`、`22`、`27`。
5. 涉及配置、第三方或初始化时，再下钻：`16`、`17`、`18`、`19`、`23`。
6. 涉及模型、状态和长期维护时，再下钻：`10`、`21`、`24`、`26`。

## 执行链路

```text
索引 -> 主题规则 -> 实现/评审 -> 证据回填 -> wiki/sdd 沉淀
```
