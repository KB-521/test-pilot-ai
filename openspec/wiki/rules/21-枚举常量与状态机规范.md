# 枚举常量与状态机规范

## 文档说明

- 覆盖规则：枚举、常量、状态字段和状态机迁移的设计与实现规范。
- 核心要求：离散值必须统一定义来源；状态流转要可追溯、可测试，禁止散落魔法字符串。
- 应用场景：定义状态字段、错误码、类型码、流程流转和权限常量时。
- 命中主题：enum、状态机、常量、离散值
- 触发关键词：enum,status,state machine,type code,magic string
- 关联 skill：python/backend-design、python/req-impl、python/test-create-unit
- 前置规则：04、10
- 后置规则：22、27
- 排除场景：纯静态展示文案修改

## 规则

1. 状态和类型字段优先使用枚举表达，不用散落字符串。
2. 需要状态迁移时，先定义允许的迁移表。
3. 新增状态值要同步更新文档、测试和接口说明。
4. 业务常量与技术常量分开管理。
