# 多租户与请求上下文规范

## 文档说明

- 覆盖规则：请求上下文、租户、组织、用户和 trace 信息的透传规则。
- 核心要求：上下文字段必须有清晰来源和生命周期，禁止在全局变量里随意堆状态。
- 应用场景：鉴权、审计、多租户查询、跨 service 调用。
- 命中主题：tenant、org、user、request context
- 触发关键词：tenant,org,user context,request state,trace id
- 关联 skill：python/backend-design、python/req-impl、python/code-review
- 前置规则：05、16
- 后置规则：19、27
- 排除场景：完全离线的纯计算脚本

## 规则

1. 当前用户、租户、组织和 trace 信息都要有明确注入点。
2. 需要多租户隔离时，service 和 repository 层都要体现隔离语义。
3. 审计字段的来源必须统一。
4. 异步任务若需要继承上下文，要显式复制，不要隐式依赖线程本地状态。
