# 返回结果封装规范

## 文档说明

- 覆盖规则：Python API 的成功响应、失败响应、分页响应和统一字段约定。
- 核心要求：同一服务的返回结构必须稳定；分页、列表和详情的字段层级不能随意漂移。
- 应用场景：设计或实现 API、统一前后端契约时。
- 命中主题：response、分页、返回结构
- 触发关键词：response wrapper,pagination,response model
- 关联 skill：python/backend-design、python/req-impl、web/task-impl
- 前置规则：06
- 后置规则：08、22
- 排除场景：纯单元测试

## 规则

1. 成功响应模型要显式建模，不返回匿名 dict。
2. 列表响应要有稳定分页字段，例如 `items/total/page/page_size`。
3. 详情与列表的公共字段尽量保持一致。
4. 错误响应结构要统一，避免一处字符串、一处对象。
