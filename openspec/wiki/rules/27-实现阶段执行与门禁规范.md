# 实现阶段执行与门禁规范

## 文档说明

- 覆盖规则：实现前准备、规则命中矩阵、契约冻结、证据回填与交付门禁。
- 核心要求：未完成规则命中矩阵、契约冻结和测试证据前，禁止把实现任务标记为完成。
- 应用场景：Python 后端实现、前端实现、重构、缺陷修复。
- 命中主题：实现门禁、证据回填、规则命中矩阵
- 触发关键词：实现阶段,门禁,规则命中,契约冻结,禁止交付
- 关联 skill：req-dev、implementation-executor、python/req-impl、web/task-impl
- 前置规则：06、12、14、22
- 后置规则：common/experience-record
- 排除场景：纯需求澄清、纯方案讨论

## 强制流程

1. 读取 `openspec/wiki/rules/INDEX.md`。
2. 建立规则命中矩阵。
3. 冻结接口或行为契约。
4. 盘点状态、类型、枚举等离散值。
5. 实现并测试。
6. 回填文档和证据。

## 交付证据

- 代码证据：变更文件和关键落点
- 测试证据：命令、结果、成功/失败路径
- 文档证据：业务/技术/规则/经验是否同步
- 风险证据：剩余风险和未闭环项

## Gate

- Gate 1：未完成规则命中矩阵，禁止编码。
- Gate 2：命中规则无证据，禁止交付。
- Gate 3：未执行测试且无阻塞说明，禁止交付。
- Gate 4：存在临时方案但未标注剩余风险，禁止交付。
- Gate 5：接口或行为变更未评估文档同步，禁止交付。
