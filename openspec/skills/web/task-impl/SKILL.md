# 前端任务实现

## 强制执行顺序

```text
需求输入
  -> 业务逻辑与字段规则对照
  -> 接口/数据层
  -> mock 或测试替身
  -> 页面与交互
  -> 代码评审
  -> 自检与交付
```

## 要求

1. 页面只能调用统一数据层，不直接散写请求。
2. 成功、失败、空态、边界行为都要覆盖。
3. 评审前不得标记完成。
4. 若已有 mock 体系，必须同步对齐 mock。
5. 字段规则要明确落到表单、请求参数或展示逻辑。

## 最终输出必须包含

- 业务逻辑对照
- 字段规则对照
- 接口或数据层文件
- 页面文件
- mock 或测试替身文件
- review 结论
- 测试或自检结果
