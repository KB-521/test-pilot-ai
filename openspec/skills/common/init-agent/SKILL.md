# 初始化项目指南

## 适用场景

- 仓库第一次接入 OpenSpec。
- `AGENTS.md` 缺失或过期。
- 技术栈、目录结构或命令发生稳定变化。

## 强制流程

1. 识别项目类型：前端、Python 后端、混合项目。
2. 扫描稳定入口：
   - 前端：`package.json`、`src/`、构建配置
   - Python：`pyproject.toml`、`requirements*.txt`、`src/`、`app/`、`tests/`
3. 只记录真实存在且可执行的命令。
4. 输出目录树和模块职责。
5. 幂等更新 `AGENTS.md`。

## 输出章节建议

1. 项目结构与模块组织
2. 构建、测试与开发命令
3. 编码规范与命名约定
4. 前端约定（如适用）
5. Python 后端约定（如适用）
6. 配置与环境
7. 测试与质量门禁
8. 文档与工作流入口

## 禁止项

- 不写临时任务状态。
- 不写敏感信息。
- 不写与仓库事实不符的示例命令。
