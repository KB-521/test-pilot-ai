# PRD 审查

## 何时使用

- 用户要求“检查 PRD”
- 需求准备进入设计阶段
- PRD 发生较大更新，需要回归审查

## 强制流程

1. 通读 PRD，提取功能点和业务对象。
2. 读取 `openspec/wiki/business/INDEX.md` 和相关业务文档。
3. 读取 `openspec/sdd/INDEX.md` 和同域历史设计或任务（若存在）。
4. 输出业务对照证据表。
5. 按 `BLOCKER / MAJOR / MINOR / SUGGESTION` 分级问题。
6. 给出下一步动作。

## 审查维度

- 结构是否完整
- 业务规则是否明确
- 状态流转是否闭环
- 权限、数据边界、异常行为是否明确
- 验收标准是否可测试
- 与历史设计或业务事实是否冲突

## 输出格式

```markdown
## PRD 审查结论
- 文档：
- 总体评级：

## 业务对照证据
| PRD 功能点 | 对照文档 | 结论 | 建议 |
| --- | --- | --- | --- |

## 问题清单
1. [BLOCKER] ...

## 下一步动作
1. ...
```
