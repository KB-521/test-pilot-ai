# 任务拆分

## 工作流

1. 输入 PRD、设计或需求卡。
2. 提取功能点和技术约束。
3. 划分批次：
   - 基础骨架
   - 核心能力
   - 配套能力
   - 治理与收尾
4. 为每个批次生成任务卡。
5. 补充依赖与并行关系。
6. 更新 `openspec/sdd/changes/<change-id>/tasks/INDEX.md`。

## 交付要求

- 每个任务都要有目标、输入、输出、验收标准。
- 依赖必须显式写清。
- 临时方案和后续动作不能混写成“已完成”。
- 拆分结果要能直接支撑实现顺序或里程碑安排。

## 推荐批次

1. 骨架与门禁
2. 核心链路
3. 配套能力
4. 测试与治理
