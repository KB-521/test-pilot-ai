# 初始化业务知识

## 路由规则

- 命中 `前端`、`页面`、`路由`：沉淀页面级业务域。
- 命中 `后端`、`python`、`接口`、`service`：沉淀服务级业务域。
- 未指定范围：默认按“前端 -> Python 后端”执行。

## 产出要求

1. 更新 `openspec/wiki/business/INDEX.md`。
2. 每个业务域要描述：
   - 目标
   - 关键流程
   - 规则与边界
   - 错误与异常行为

## 建议粒度

- 前端：按页面、路由或用户任务流
- Python 后端：按服务域、接口组或核心流程
- 跨前后端：按统一业务域拆一篇主文档，再补技术文档承接

## 约束

- 业务事实优先来自代码、设计和验收口径。
- 没有证据时，不要猜业务规则。
